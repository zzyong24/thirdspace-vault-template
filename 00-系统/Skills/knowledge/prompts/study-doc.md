### 学习文档生成指引

你的任务是把高密度内容转化为一份**完整的学习文档**，目标：用户凭这份文档就能回忆和引用核心内容，无需再回去看原文。

### 输出结构（严格按此顺序）

1. **内容概览**
   - 一句话总结（≤30 字）
   - 类型：访谈 / 教程 / 播客 / 文章 / 文档
   - 篇幅：字数或时长
   - 难度：入门 / 进阶 / 高级
   - 目标读者
   - ⭐ 核心结论（学完你能带走的一句话）

2. **完整结构**
   按原文逻辑顺序，用层级标题（H2/H3）还原全文结构。每个主要章节用一句话说明「这一节讲了什么、为什么重要」。

3. **精华摘录**（可直引用的原话，带章节来源标注）
   选 3~7 个最值得引用的原段落，原文照录，不改写、不概括。

4. **知识地图**（必须包含 Mermaid 图表）
   必须包含以下两类图表：
   - **概念关系图**（`graph LR/RL/TB`）：展示核心概念之间的关联、因果或层次关系
   - **流程/步骤图**（`flowchart` 或 `sequenceDiagram`）：如果内容涉及过程、方法论或决策流，必须用图还原

   ```mermaid
   graph TB
       A[核心概念] --> B[分支概念]
       B --> C{关键判断}
       C -->|是| D[结论1]
       C -->|否| E[结论2]
   ```

   图表下方附「关键洞察清单」（编号列表）：
   - 洞察 1：（洞察内容 + 来自原文哪个观点）
   - 洞察 2：...
   - 与已有知识的连接：...

5. **实践指南**
   - 适合谁用、什么场景最匹配
   - 怎么用（具体的操作步骤或思考框架）
   - 常见误区或陷阱

6. **延伸学习**
   - 相关推荐（文章/视频/工具），每项附一句话推荐理由
   - 关键词（用于后续搜索）
