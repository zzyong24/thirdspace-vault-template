# 工作区工具框架与渐进加载规范

## 目标

ThirdSpace 不再把 CLI 当成人类入口。所有能力对外表现为 Skill，脚本只作为 Skill 内部实现。

所有 Skill 的 canonical 根目录是：

```text
00-系统/Skills
```

Hook、crontab、Agent 自动化任务规格的 canonical 根目录是：

```text
00-系统/运行时
```

## 三层加载

1. 根 Skill：`thirdspace-vault`
   - 解析 vault。
   - 判断工作区。
   - 路由创建文件。
   - 记录工作日志事件。
   - 执行全局审计。

2. 工作区 Skill：`workspace-*`
   - 读取当前工作区 `WORKSPACE.md`。
   - 约束文件创建、命名、Frontmatter、状态和流转。
   - 维护本工作区自治边界。

3. 领域 Skill
   - 只在意图命中时加载。
   - 例如 `lifeos` 处理人际事件，`worklog` 处理工作日志，`article` 处理文章输出。

## 工具框架契约

机器可读配置见：

```text
.thirdspace/schema/workspace-tools.yaml
```

每个工作区必须声明：

- workspace skill
- domain skills
- 可用工具动作
- legacy source paths
- 创建、更新、审计、流转能力

## 旧 CLI 迁移原则

- 旧 CLI 命令不再直接暴露给用户。
- 仍有价值的脚本迁入对应 Skill 的 `scripts/`。
- Prompt 迁入对应 Skill 的 `prompts/`、`templates/` 或 `references/`。
- 旧路径只允许出现在 `legacy_sources` 和迁移队列中。
- 已迁移旧路径只允许出现在 `archived_legacy_sources`、manifest 和归档报告中。
- `.codex/skills`、`.claude/skills`、`.cursor/skills` 和项目内局部 Skill 不再作为当前维护入口；有价值内容必须迁入 `00-系统/Skills`。
- 换电脑初始化时，Agent 必须调用 `install-runtime --all` 从 `00-系统/运行时` 恢复本机 hook 和 crontab。

## LifeOS 子系统

LifeOS 作为领域 Skill，不再作为根目录结构存在：

- 人际事件：`02-日记/人际事件/`
- 人物档案：`05-资源/人物档案/people.json`
- 机器副本：`.thirdspace/data/lifeos/people.json`

Agent 处理“人际、人物、关系、事件复盘、决策模拟”时，必须在 `workspace-journal` 或 `workspace-resources` 之后渐进加载 `lifeos`。

## Flux 输入流

旧 `flux/intake` 已迁移为：

- Markdown：`01-收件箱/网页剪藏/`
- 图片资产：`05-资源/图片/flux-intake-assets/`
- JSON 等数据附件：`05-资源/附件/flux-intake-data/`

迁移 manifest：

```text
.thirdspace/manifests/20260525_flux-intake-migration.json
```

新内容不得再写入 `flux/intake`。
